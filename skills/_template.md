# Skill Template

## Purpose
Describir una capacidad reusable para tareas instruccionales específicas.

## When to Use
- [ ] Frases o contextos de activación documentados.

## Inputs
- Contexto de dominio
- Objetivo instruccional
- Restricciones operativas

## MVP Limits
- Delimitar explícitamente qué NO hace esta skill en v0.1.0.
- Mantener salida markdown-first, sin ejecución runtime ni integración real.

## Outputs
- Artefacto markdown breve y verificable con trazabilidad a evidencia.
- Incluir criterios mínimos de aceptación observables.
- Si falta información crítica, usar placeholders explícitos (`[POR DEFINIR]`, `[SUPUESTO]`).

## Process
1. Validar contexto y restricciones.
2. Construir salida mínima, verificable y trazable.
3. Declarar límites MVP, supuestos controlados y pendientes de validación.

## Verification Notes
- [ ] Trigger y contexto de uso definidos.
- [ ] Entradas/salidas mínimas especificadas.
- [ ] Proceso mínimo y límites explicitados.

# Skill: Pre/Post Test Builder

## Purpose
Diseñar medición pre/post para comparar progreso de aprendizaje.

## When to Use
Cuando se necesita línea base y medición final comparable.

## Inputs
- Objetivos de aprendizaje.
- Nivel esperado de desempeño.
- Restricciones de aplicación de prueba.

## Outputs
Diseño comparativo pre/post con blueprint por dimensión, equivalencia entre pre y post, rúbrica de interpretación y estructura mínima de ítems.

## MVP Limits
- Diseña la lógica comparativa entre prueba inicial y final.
- No construye bancos amplios para Moodle (`moodle-question-bank-builder`).
- No exporta preguntas.
- No sustituye rúbricas de desempeño práctico.

## Process
1. Definir blueprint (objetivo/dimensión → cobertura de ítems y peso).
2. Asegurar equivalencia de dificultad, formato y cobertura entre pre y post.
3. Diseñar ítems con distractores plausibles y técnicamente defendibles.
4. Definir rúbrica e interpretación por dimensión (avance, estancamiento, alerta).

## Verification Notes
- [ ] Cobertura objetivo→ítem clara.
- [ ] Comparabilidad pre/post garantizada.
- [ ] Distractores plausibles y no triviales.
- [ ] Blueprint explícito y verificable.
- [ ] Rúbrica e interpretación por dimensión definidas.

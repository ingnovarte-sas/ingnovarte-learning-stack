# Workflow: LDD Spec

## Objetivo
Formalizar requerimientos instruccionales verificables.

## Activación orgánica
- Este workflow puede ser activado por el `learning-orchestrator` cuando la solicitud lo amerite.
- El usuario no necesita invocarlo manualmente ni conocer la fase.

## Inputs
- Dirección recomendada de `ldd-explore`.
- Objetivos de negocio/operación del curso.
- Restricciones de canal (PPT, Moodle, SCORM, Captivate).

## Pasos
1. Definir objetivos y resultados esperados.
2. Especificar escenarios de uso y evidencia.
3. Registrar límites técnicos y de canal.

## Quality Gates
- [ ] Requerimientos claros y testeables.
- [ ] Criterios de evidencia definidos.

## Exit Criteria
- [ ] Especificación lista para diseño técnico.
